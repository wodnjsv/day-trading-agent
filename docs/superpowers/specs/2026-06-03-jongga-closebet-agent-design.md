## 1. 목표와 비목표

**목표**
- 하루 1회 **시황·수급·재료·테마·차트**를 분석해 **유동성 두꺼운 대장주 중심**으로 종가에 살 바스켓을 선정한다.
- 종가에 자동 매수 → **익일 시초가에 (귀속분) 전량 자동 매도**한다 (오버나잇 갭 수익).
- 매 회차 **근거 리포트**를 남겨 복기 자산으로 축적한다.
- "LLM은 분석·선정만, 주문 방아쇠와 리스크 한도는 룰"이라는 분리로 가드레일을 견고하게 한다.

**핵심 설계 원칙**
- **선정 신호는 'as-of 안전(t-1 확정)' 팩터를 기준선으로 한다.** 백테스트는 라이브를 *근사*하되, **동시호가 마지막 수분 드리프트는 EOD 데이터로 복원 불가능한 잔여 look-ahead**다 — "백테스트=라이브"는 완전 달성 불가이며 §8.4에서 측정·관리한다(은폐 금지).
- **검증의 비대칭을 인정한다.** historical 백테스트는 "기계적 바닥 + 정량 팩터 엣지"만 검증. 진짜 알파인 **LLM 선정층은 historical 불가** → **포워드 A/B(LLM shadow vs 룰 baseline)로만** 검증되며, 이것이 LLM 실돈 점등의 게이트다(§8.5).
- **실돈은 검증 통과분에만.** 룰 baseline(historical 통과)만 소액 LIVE, LLM arm은 A/B 통과 전 shadow. 실돈 arm엔 누적손실 킬을 사전등록한다.

**비목표 (YAGNI)**
- **시가배팅·배수배팅·갭다운 홀딩** — Phase 2 별도 모듈. (마감 전 공개 악재 진입 회피 + 장전 재점검 알림은 MVP.)
- 장중 틱/호가 실시간 엔진, 멀티 전략 프레임워크, 딥러닝 모델 훈련(차트 수식은 결정론적 팩터로만).
- 분산 인프라(메시지버스/상시 데몬).

## 3. 아키텍처 & 컴포넌트

**4-시점 배치.** 상시 데몬 아님.

```
══ Pass 1 — 장중 (≈14:00): 사전 리서치 ════════════════════════════════
 1. MarketRegime     t-1 확정 지수·시장거래대금 → 시황등급 + 비중배수         [룰]
 2. PreScreener      t-1 확정 데이터로 광의 후보군(예 100~150) 압축           [룰]
 3. CatalystResearch 후보별 뉴스 + LLM 웹검색 → 재료·테마(스냅샷 영속화)      [LLM+도구]

══ Pass 2 — 동시호가 직전 (≈15:20~15:28): 확정·실행 ════════════════════
 4. FinalScreener    t-1 안전팩터 랭크 + 15:2x 예상체결 veto(급락/거래정지)    [룰]
 5. Selector         시황+후보(팩터)+재료 → 바스켓·근거·conviction(랭킹용)     [LLM/룰 fallback]
 6. RiskGovernor     정량가중 sizing + 한도·호가수용cap·악재·블랙리스트 심사     [룰, 거부권]
 7. CloseExecutor    승인 바스켓 종가 동시호가 매수 (reconciliation·15:28 데드라인) [KIS 주문]
 8. ReportWriter     시황·후보·재료·근거·갭다운시나리오·체결 기록              [리포트]

══ 익일 장전 (≈08:30~09:00): 재점검 ═══════════════════════════════════
 9. PreOpenCheck     귀속분 거래정지/하한가/오버나잇 악재 점검 → 강알림        [룰+알림]

══ 익일 시가 (≈09:00) ═════════════════════════════════════════════════
10. OpenExecutor     실잔고 ∩ 귀속분 시초 전량매도 + 실현손익 기록            [KIS 주문]
11. (다음 회차 1·3·5에 직전 복기 컨텍스트 제공)
```

> **후보군 일관성:** Pass 2 최종 후보는 Pass 1의 **strict 부분집합**. PreScreener는 t-1 거래대금 기준 **광의**로. 14:00~15:2x 장중 신규 부상 종목은 **MVP 범위 밖(로그)** — 대형주 중심이라 영향 제한, t-1 lag 편향은 알려진 한계(Phase 2 보강).

### 핵심 데이터 계약 — `DailyPlan`
분석 단계가 쓰고 실행 단계가 읽는 단 하나의 계약. **필드별 생산 단계 명시.**

```
DailyPlan
 ├ date, as_of_signal(=t-1), as_of_exec(=15:2x 예상체결 스냅샷 시각)   # 메타(1·4 생산)
 ├ regime: { 등급(BULL/NEUTRAL/BEAR/HALT), 비중배수 }                # 1 생산(t-1)
 ├ basket[]: {
 │    symbol,
 │    factors(거래대금·수급·정배열·신고가·거래량비·지지),             # 4 생산(전부 t-1 안전)
 │    catalyst(요약·강도·신선도·persistence·출처), theme,             # 3 생산
 │    rank, conviction(0~1, 랭킹·기록용 — 사이징 미사용),             # 5 생산(룰 fallback=팩터점수)
 │    rationale, gap_down_scenario,                                   # 5 생산(거버너 진입회피 참조)
 │    weight, target_qty,                                            # 6(Sizer, 정량가중) 생산
 │    accepted_qty, filled_qty, avg_price }                          # 7(발주 reconcile·체결) 갱신
 ├ risk_limits: { 종목당상한, 총익스포저, 호가수용cap, 블랙리스트, 악재플래그 }
 ├ attribution: { intended_symbols, intended_qty(체결확정 시 동결) }  # 익일 청산 범위 한정
 └ report_path
```

- **영속화:** SQLite. CatalystResearch 웹검색 결과도 **스냅샷 영속화**(시점통제·복기).
- **정합성 원칙:** 익일 매도(10)의 **청산 범위 = `실잔고 ∩ attribution`**. Plan=의도, 잔고=실제, 귀속=교집합.

### 컴포넌트 분리 원칙
- 분석·실행은 `DailyPlan` 하나로만 통신.
- `RiskGovernor`는 실행과 분리된 게이트(거부권).
- **데이터 어댑터(`MarketDataProvider`)**가 KRX(pykrx)/KIS에 묶이는 유일한 곳. 두 소스가 같은 의미의 수치를 내도록 정규 스키마 + 골든 테스트(§8.3).

## 4. 데이터 흐름 (하루 사이클)

```
Pass 1 (≈14:00)        ① 시황등급(t-1) → ② 광의 후보(t-1) → ③ LLM 재료조사(스냅샷)
Pass 2 (≈15:20~15:28)  ④ t-1 안전팩터 랭크 + 예상체결 veto → ⑤ 바스켓 선정
                       → ⑥ 정량가중 sizing + 심사 → ⑦ 종가 매수(reconcile) → ⑧ 리포트
익일 장전 (≈08:30)      ⑨ PreOpenCheck(거래정지·하한가·악재 알림)
익일 시가 (≈09:00)      ⑩ 실잔고∩귀속분 시초 전량매도 → 손익 → 복기
```

### 4가지 핵심 규칙
1. **방아쇠는 항상 룰.** LLM은 ③⑤만. 실행·한도·사이징은 룰.
2. **결정 시점 = 가용 정보만.** 선정 팩터·수급 = t-1 확정. 15:2x 예상체결은 실행가·veto·cap 분모에만. 확정 종가(미래값) 선정 참조 금지(잔여 드리프트 §8.4 측정).
3. **출구는 귀속분만 무조건.** 시초 전량매도는 `실잔고 ∩ attribution`, 손익·갭 무관(MVP).
4. **잔고가 진실.** 실행은 KIS 실잔고·미체결주문에서 동기화하고 시작.

## 5. 종목 선정 상세

### 5.1 Universe (모집단) — 유동성 중심
만쥬 1순위 = **호가창 두께**.
- **1차 하드필터(되돌릴 수 없는 제거 최소):** ① 거래대금 상위 K ∩ ② 호가수용력 ∩ ③ 거래정지·관리·정리매매 제외 ∩ ④ 동전주·상폐임박 배제 수준 **매우 낮은 안전 시총 하한**.
- **대형주는 하드컷 아닌 자연 수렴:** 거래대금·호가수용 cap × 비중으로 충족 → 거래대금 충분한 중소형 테마 대장주 선제배제 안 함.
- 시총은 랭킹 팩터(하드 AND 아님). 컷은 §12.1.

### 5.2 Stage 4 — 스크리너 팩터 (결정론적, **t-1 안전**)
**as-of:** 선정 팩터는 전부 **t-1 확정**. 오늘 강도 의존 신호는 백테스트 불가·낙관편향원 → 선정 미사용, 15:2x 예상체결은 **veto + 실행가 + cap 분모**에만. 누수 방지: 롤링 통계 t-1까지만. 정상성: 비율·거리·로그수익률.

**① 거래대금** — t-1 절대 + 시장 상대순위.
**② 수급** — 외국인·기관 순매수(t-1 확정 + 최근 5일 누적). 분모 = **동일 t-1 윈도우 거래대금**. 카테고리는 §8.3 정규 스키마.
**③ 차트 팩터(t-1 종가):**

```
정배열  Spread_t      = (MA_short − MA_long) / MA_long
        Alignment     = 1.0 if MA5>MA20>MA60>MA120 ; else (충족 쌍 수 / 3) 보간
신고가  Proximity_N   = Price_{t-1} / max(Price_{t-1..t-N})
        DaysSinceHigh = (t-1) − argmax(Price_{t-1..t-N})        # 소진 모멘텀 패널티로도 사용(§5.5)
거래량  VolRatio      = mean(양봉일 거래량)/mean(음봉일 거래량)
지지    NearMA        = Price_{t-1}가 MA5/MA20 ±x% 이내 + 직전 반등
```

**랭킹:** 하드필터 통과분 팩터 가중합 → 상위 N. 가중치·룩백 N은 §12.1.

### 5.3 Stage 3 — CatalystResearch (LLM + 도구)
`{ has_catalyst, summary, strength, recency, persistence, theme, sources[] }`. **persistence** = *이미 가격 반영* vs *미반영 잔존 모멘텀* 구분. 검색결과 스냅샷 영속화. 시스템 프롬프트 캐싱.

### 5.4 Stage 5 — Selector (LLM, 구조화 출력)
입력: 시황등급 + 후보(t-1 팩터) + 재료. 출력: `{ symbol, conviction, rationale, theme, gap_down_scenario }`. conviction은 **선정·랭킹·기록용**, **사이징 미사용**(#15). gap_down_scenario는 기록 + ⑥ 진입회피 참조(홀딩 Phase 2).

### 5.5 룰 baseline Fallback Selector
백테스트 룰 baseline은 LLM(3·5) 꺼짐:
- 선정 = §5.2 팩터 가중합 상위 K / conviction = 정규화 팩터점수 / weight = §6 정량가중.
- **persistence 프록시:** baseline에 DaysSinceHigh 패널티 팩터 반영(소진 모멘텀 매수 방지). catalyst·theme 공란. → "재료 없는 정량 팩터만의 엣지" 측정. §8.4-1 IC는 선정 꼬리(상위 N) 구간도 평가.

## 6. Sizing & 리스크 거버너

```
종목 비중 = 자본 × 시황배수 × 정량가중(균등 또는 팩터점수 비례)
   cap by  종목당상한 · 총익스포저상한 · 호가수용cap
```

- **사이징은 정량가중만.** LLM conviction은 선정·랭킹·기록에만, 사이징 미사용(미보정 0~1을 포지션 크기로 직결 금지). Phase 4 보정 후 도입 검토. → **라이브 사이징 경로 = 백테스트 경로.**
- **시황배수:** BULL 1.0 / NEUTRAL 0.5 / BEAR 0.2 / HALT 0. + 강세확신 tier(예 STRONG_BULL 1.5) — 항상 cap 종속.
- **호가수용 cap(#B/#ER-N3):** 분모 = **내 주문 제외 잔여 예상체결량(ex-ante, 발주 직전 스냅샷)**. 막판 유입(15:28~15:30) 보수 디스카운트. 백테스트는 EOD proxy(§8.2)와 의미차를 정합성 점검에 등록. 라이브 산출 = KIS 예상체결 TR(Phase 1 어댑터 게이트).
- **거버너 거부권:** HALT→0 / 한도초과→감액·제외 / **마감 전 공개 악재공시 감지→진입거부** / **gap_down_scenario상 과도위험→진입거부·감액** / 블랙리스트→제외.
  - ⚠️ **악재필터 한계:** 15:28까지 공개분만. DART 악재 상당수는 마감 후(16~18시)라 **오버나잇 최대 리스크는 진입필터로 못 막음** → 종목당 사이징·바스켓 분산·익일 시초 청산으로 완화, 장전 ⑨ 감지·알림.
- **매도는 막지 않음.**

## 7. 실행 & 정합성

### 7.1 Stage 7 — 종가 매수 (reconciliation 기반)
- **종가 동시호가(15:20~15:30) 매수**, **15:28 발주 데드라인**.
- **발주 멱등(#ER-N2):** 동시호가 단일가는 15:30 전 체결 미확정 → "체결조회 후 재시도" 불가. 재시도/추가발주 직전 **KIS 미체결주문조회로 접수 주문량(미체결 포함) 합산 → `target − accepted_qty`만 추가 발주.** 멱등 기준 = filled_qty 아닌 `accepted_qty`. 응답 유실 시 '미발주' 가정 금지 → 주문내역 재조회.
- 15:30 체결 확정 후 `filled_qty/avg_price` 갱신, `attribution.intended_qty` 동결.

### 7.2 Stage 9 — 장전 재점검 (PreOpenCheck)
- 익일 **08:30~09:00**, 귀속분: 거래정지/하한가, 오버나잇 DART 공시·뉴스 스캔 → **강알림**.
- MVP는 매도 결정 불변(시초 청산). 거래정지/하한가는 ⑩ 청산불가 분기로 연결. Phase 2 시가배팅 훅.

### 7.3 Stage 10 — 시초 매도
- **KIS 실보유 잔고 조회 → `실잔고 ∩ attribution`만 시초 전량매도.** 실현손익 기록.
- **전용 계좌 권장.** attribution 교집합은 비전용 환경에서도 무관 보유분 보호(잔고에 있으나 attribution 없으면 청산 안 함·알림).
- **시초 청산 불가(하한가/거래정지):** 미체결 → 추적·재시도·강알림, 사실상 강제 다일 보유(모델링 §8.2).

| 상황 | 대응 |

### 8.2 엔진 (as-of 강제 + 시장충격 + 강제다일)
- 매 거래일 `종가매수 → 익일시초매도` 시뮬.
- **as-of:** 선정 팩터·수급 = t-1 확정. 매수가 = t 종가(실행). t 종가 ≈ 15:2x 예상체결 — **마지막 수분 드리프트는 EOD 복원 불가 잔여**로 §8.4 민감도 측정.
- **체결모델(#B/DF-A):** 무충격 폐기. **시장충격/슬리피지 함수(매수·매도 대칭, 갭다운 가중).** 입력의 **양면 정의:** 백테스트 = EOD 산출가능 proxy(일거래대금 대비 주문비중 + 시장평균 동시호가비중, 또는 **보수 상수 슬리피지 밴드**); 라이브 = KIS 예상체결 TR. "동시호가 가용물량"은 라이브 전용. 충격 파라미터는 점추정 아닌 **보수~낙관 밴드**로 §8.4 민감도 사전등록. Phase 3/4 라이브 자기주문 슬리피지 캘리브 루프.
- **강제 다일(m1/BV-4):** 시초 하한가/거래정지 청산불가는 **정량 규칙**(연속 하한일수·재개 첫날 갭 p25 등 데이터 기반, §12.1 동결)으로 후속 불리 드리프트 보수 반영. hand-wave 금지.
- 메트릭: 승률·평균갭수익·MDD·회전율, **net**(수수료 + 거래세 0.18~0.23% + 슬리피지 밴드).

### 8.3 데이터 소스 (어댑터 + 정규 스키마)
| 데이터 | 백테스트(pykrx EOD) | 라이브(KIS) |
|---|---|---|
| 일봉 OHLCV·거래대금 | ✓ | ✓ |
| 투자자별 수급(t-1) | ✓ (정규 스키마) | ✓ |
| 시총·상장주식수 | ✓ | ✓ |
| 상장/상폐 목록 | ✓ (생존편향) | — |

### 8.4 검증 게이트 — 사전 등록
파라미터 탐색 *전*에 합격선·프로토콜 등록. walk-forward(롤링) + 최종 홀드아웃(1회만).
1. **[1순위·load-bearing, 룰 baseline 한정] 팩터 1박 갭 IC:** close→open 수익 정보계수. **다중검정 보정(Bonferroni/BH) + 보정후 α + 유의 팩터 ≥ m개 + 부호 안정성** 사전등록(BV-1). 실패 → **"룰 baseline 가설 붕괴"(baseline 비활성)** — 전략 전체 중단 아님(§8.5와 결합, C3).
2. **[바닥] 비용 net 엣지:** 충격·슬리피지는 §8.2 **밴드로 보수~낙관 동시 보고**(DF-A). MDD·최소 표본·승률 한도.
3. **[드리프트 민감도] t-1 vs (참고) t-종가 선정 비교:** **정량 컷오프 θ1(자카드/회전)·θ2(수익델타) 사전등록**(BV-5), 초과 시 아티팩트.
- 다중검정: 탐색은 walk-forward 내부, 홀드아웃 1회만.

### 8.5 검증 아키텍처와 A/B 게이트 완결 (핵심 — SC-1·BV-3·C3)
- historical(§8.4)은 룰 baseline의 바닥·팩터엣지만. **진짜 알파(LLM)는 포워드 A/B로만.**
- **arm 배정(공정·안전 동시):**
  - **룰 baseline:** historical 통과분 → **소액 실거래 LIVE**(배관·바닥 포워드 + 충격함수 라이브 캘리브).
  - **LLM arm:** A/B 기간 **shadow(페이퍼)**, 룰 arm과 **동일 시장충격 모델**로 기록.
  - **A/B 비교** = paper-LLM vs paper-rule(동일 충격모델 → 사이징·지면 비대칭 제거). **같은 날·같은 총자본·같은 사이징·같은 PreScreener.** 종목 중복은 **paired 검정**. 비중복 표본 **최소 검정력·표본기간 사전등록**.
- **손실 한도(SC-1):** 실돈인 룰 baseline arm에 **누적손실 조기중단(킬) 임계치 사전등록.** LLM arm은 shadow라 실돈 무노출.
- **LLM 실돈 점등:** paper A/B가 사전등록 우위(net·검정력) 통과 후에만 LLM 실거래 전환(Phase 3b).
- **전략 전체 중단(C3):** §8.4-1 IC 실패는 baseline 붕괴만. 전략 전체 중단은 **baseline·LLM A/B 둘 다 실패** 시(바닥 0이어도 LLM A/B net 양수면 조건부 진행).
- **정직:** 핵심 thesis는 포워드 A/B 전까지 검증 안 됨 — 이 비대칭을 알고 Phase를 설계.
